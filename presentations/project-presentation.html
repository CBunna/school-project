Information Architecture

Site Structure

The website is organized into 5 main pages with clear hierarchy and logical navigation flow

🏠 Home Page

Hero section, about Beskydy, features (hiking, cycling, culture), top attractions preview, upcoming events, CTA sections

🏔️ Attractions

Main attractions with detailed info (Lysá Hora, Pustevny, Radhošť), additional destinations, GPS coordinates, amenities

🏨 Accommodation

Hotels, pensions, cottages, apartments with pricing, ratings, amenities, booking information

⛷️ Activities

Summer activities (hiking, biking, water), winter activities (skiing, sledding), cultural events, gastronomy

📬 Contact

Contact form, office info, emergency contacts, transportation guide, location map, FAQs

🧭 Navigation

Sticky header, mobile hamburger menu, footer links, social media, skip-to-content link

Usability Features

Design Principles
  • Consistency: Uniform design across all pages with consistent navigation, colors, and typography
  • Clarity: Clear headings, readable fonts (16px base), high contrast text
  • Feedback: Hover states on buttons/links, form validation messages, loading indicators
  • Simplicity: Clean layout, minimal distractions, focused content
  • Efficiency: Quick access to information, fast page loads, optimized images
Mobile-First Approach
  • Responsive breakpoints: 480px, 768px, 1024px
  • Touch-friendly buttons (minimum 44×44px)
  • Hamburger menu for mobile navigation
  • Optimized images for different screen sizes

Accessibility Standards

WCAG 2.1 Compliance

The website follows Web Content Accessibility Guidelines to ensure all users can access and use the content

Semantic HTML
  • Proper heading hierarchy (h1 → h2 → h3)
  • Semantic elements (header, nav, main, section, footer)
  • ARIA labels and roles for screen readers
  • Alt text for all images
  • Skip-to-content link for keyboard users
Keyboard Navigation
  • All interactive elements accessible via Tab key
  • Focus indicators (3px outline, high contrast)
  • Logical tab order
  • No keyboard traps
Structured Data
  • Microformats (hCard, hProduct, vevent)
  • Meta tags for SEO and social media
  • Open Graph protocol
Visual Accessibility
  • High contrast ratios (WCAG AA compliant)
  • Readable font sizes (minimum 16px)
  • Sufficient line spacing (1.6 line-height)
  • Color not sole indicator of information
  • Scalable text (responsive units: rem, em)
Form Accessibility
  • Labels associated with form inputs
  • Required field indicators
  • Clear error messages
  • Fieldset and legend for grouping
  • Autocomplete attributes
Testing Tools Used
  • W3C HTML/CSS Validators
  • WAVE accessibility checker
  • Lighthouse audit (Chrome DevTools)
  • Screen reader testing

Technical Implementation Examples

CSS Grid Layout

Used for attraction cards, accommodation grid, and feature sections. Automatically adapts to different screen sizes with auto-fit and minmax.

CSS Flexbox

Implemented in navigation menu, hero content centering, footer layout, and card internal structure for alignment.

CSS Custom Properties

Color scheme, spacing scale, typography, and transitions defined as CSS variables for easy theming and maintenance.

JavaScript Features

Intersection Observer API for lazy loading and scroll animations, form validation with regex, smooth scroll behavior.
