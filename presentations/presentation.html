What We Built Last Semester

A complete frontend for a Beskydy Mountains tourism guide

  • 5 pages: Home, Attractions, Accommodation, Activities, Contact
  • Responsive design — works on mobile, tablet, desktop
  • Accessible UI with semantic HTML and ARIA labels
  • Client-side form validation on all forms

What is still missing

  • Forms show a success message but save nothing
  • All content is hardcoded directly in HTML
  • The API function exists in the code but points nowhere
  • No database — no real data storage

Goals This Semester

🖥️

Server

Build a Node.js + Express server that handles all requests from the frontend.

🗄️

Database

Use PostgreSQL to store attractions, accommodation, activities, and contact messages.

🔗

REST API

Create endpoints that the frontend can call to get real data instead of hardcoded HTML.

📬

Forms

Make the contact form actually send and save messages to the database.

How It Will Work

Expected Result

Before (Now)

  • Static HTML with hardcoded content
  • Forms show a message but save nothing
  • No database, no server
  • Data cannot be updated without editing HTML

After (Goal)

  • Content loaded dynamically from the database
  • Contact form saves real messages
  • Node.js server + PostgreSQL database running
  • Data can be updated without touching HTML
